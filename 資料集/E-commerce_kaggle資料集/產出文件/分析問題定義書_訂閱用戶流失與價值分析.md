### **分析問題定義書**

**1. 核心問題**
> 哪些訂閱用戶具有較高的流失或暫停風險？這些用戶在人口屬性、訂閱行為與購買行為上具備哪些可被量化的共同特徵，可作為提前預警與精準挽回行動的依據？

**2. 分析目標**
- **目標一**：在 5 個工作天內，基於現有資料集，識別 `subscription_status` 為 `cancelled` 或 `paused` 用戶在人口屬性（年齡、性別、國家）與行為特徵（`cancellations_count`、`purchase_frequency`、`preferred_category`、實際購買 `category`）上的共同模式，產出高流失風險用戶畫像。
- **目標二**：量化不同訂閱狀態群體（`active` / `paused` / `cancelled`）在平均訂單價值（AOV）、購買頻率與取消次數上的差異，評估不同狀態對整體營收貢獻與風險的影響。
- **目標三**：計算各國家市場與主要品類（`category`）的用戶流失率與暫停率，找出流失風險最高的「國家 × 品類」組合，作為後續市場與品類策略調整的優先順序。

**3. 關鍵指標**

- **主要指標**：
  - 訂閱流失率（Churn Rate）  
    `Churn Rate = (subscription_status = "cancelled" 的用戶數) / (總用戶數)`
  - 訂閱暫停率（Pause Rate）  
    `Pause Rate = (subscription_status = "paused" 的用戶數) / (總用戶數)`

- **次要指標**：
  - 平均訂單金額（Average Order Value, AOV）  
    `AOV = mean(unit_price × quantity)`
  - 購買頻率（Purchase Frequency）= `purchase_frequency`
  - 用戶取消次數（Cancellations Count）= `cancellations_count`
  - 距最後購買天數（Days Since Last Purchase）  
    `Days Since Last Purchase = 分析基準日 − last_purchase_date`
  - 活躍訂閱比例（Active Subscription Rate）  
    `Active Rate = (subscription_status = "active" 的用戶數) / (總用戶數)`

**4. 初步假設**
- **假設一（取消次數與流失風險）**：`cancellations_count ≥ 3` 的用戶，其 `subscription_status` 為 `cancelled` 或 `paused` 的比例，顯著高於 `cancellations_count < 3` 的用戶，代表重複取消行為是高度流失風險訊號。
- **假設二（購買頻率與單價結構）**：`purchase_frequency` 偏低（例如處於整體分佈的最低 25 百分位）且 `unit_price` 偏高（例如位於最高 25 百分位）的用戶，更容易出現 `paused` 或 `cancelled` 狀態，顯示「高單價、低頻次」的消費模式與流失風險正相關。
- **假設三（國家市場差異）**：Pakistan、India 等新興市場的用戶，在控制年齡與性別之後，其流失率（`subscription_status = cancelled`）與暫停率（`subscription_status = paused`）高於 USA、UK、Germany 等成熟市場，反映支付行為與市場成熟度的差異。
- **假設四（偏好與實際購買落差）**：`preferred_category` 與實際購買 `category` 長期不一致的用戶，其流失率高於偏好與實際高度一致的用戶，代表「品類體驗與預期不符」可能導致不滿與流失。

**5. 所需數據**
本分析所需欄位均來自現有 E-commerce Customer Insights and Churn Dataset，對應如下：

- **用戶屬性**：
  - `customer_id`：用戶唯一識別碼（可用於彙總多筆訂單至用戶層級）
  - `age`：用戶年齡（可分群為不同年齡層）
  - `gender`：用戶性別
  - `country`：用戶所屬國家／市場
  - `signup_date`：用戶首次註冊日期（可推估用戶在平台上的存續時間）

- **訂閱行為**：
  - `subscription_status`：訂閱狀態（`active` / `paused` / `cancelled`），為判斷流失與暫停的核心欄位
  - `cancellations_count`：用戶累積取消次數，作為流失風險強度的重要指標
  - `last_purchase_date`：最後一次購買日期，用於計算距最後購買的天數與近期活躍度

- **訂單交易**：
  - `order_id`：訂單編號（用於交易層級分析與訂單去重）
  - `order_date`：訂單日期，可用於構建時間序列與觀察季節性
  - `unit_price`：單價
  - `quantity`：購買數量  
    → 衍生欄位：  
    `total_order_value = unit_price × quantity`  
    作為單筆訂單金額，進一步可彙總為用戶層級的總消費與平均訂單金額。

- **消費偏好與品類結構**：
  - `preferred_category`：用戶自陳或系統識別的偏好品類
  - `product_name`：商品名稱，可用於細緻品類或暢銷品分析
  - `category`：實際購買品類（如 Sports、Beauty、Electronics、Home、Clothing 等），可用於國家 × 品類的流失率分析與營收結構分析

- **購買習慣**：
  - `purchase_frequency`：購買頻率指標，用於衡量用戶的活躍度與黏著度

> ⚠️ 注意：`preferred_category`（用戶偏好）與 `category`（實際購買）可能存在差異，建議透過交叉分析判斷「偏好—實際」是否對齊，找出體驗不符導致流失的潛在風險族群。

**6. 預期產出與商業價值**
- **預期產出**：
  1. 高流失風險用戶畫像報告：包含年齡、性別、國家、取消次數、購買頻率、偏好／實際品類等維度的特徵總結。
  2. 不同訂閱狀態（`active` / `paused` / `cancelled`）在 AOV、購買頻率、取消次數與國家／品類分佈上的對比分析表與視覺化圖表。
  3. 各國家 × 品類的流失率與暫停率熱力圖，標記高風險市場與品類組合。
  4. 偏好與實際購買不一致用戶群的行為分析報告，作為產品體驗與推薦策略調整的依據。

- **商業價值**：
  - 協助行銷團隊針對高風險用戶（例如 `cancellations_count ≥ 2` 或近期 `Days Since Last Purchase` 顯著拉長者），設計個人化召回活動（如針對 `paused` 用戶的限時折扣、針對特定品類的加值方案），預期在未來 30–60 天內提升再啟動率與回購率。
  - 支援訂閱方案與定價團隊，針對高流失風險的「國家 × 品類」組合調整訂閱機制（如試用期長度、綁約條件、贈品或加值內容），以降低整體流失率與暫停率。
  - 為產品與推薦系統團隊提供「偏好—實際購買落差」的量化證據，優化推薦邏輯，減少用戶對內容或品類的不滿，進而提升用戶滿意度與長期留存。
  - 整體目標：在既有訂閱基礎上，透過針對性行動，於下一個營運季度內將高風險用戶的流失率降低 X%（可在後續根據實測結果設定具體數值），並提升整體訂閱用戶群的平均訂單價值與購買頻率。
