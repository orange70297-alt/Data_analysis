# 商業場景模擬案例 (Based on Online Retail Dataset)

這份文件包含三個基於 `Online Retail` 數據集 (2009-2010) 設計的 MBA 級別商業案例。
該數據集包含：`InvoiceNo`, `StockCode`, `Description`, `Quantity`, `InvoiceDate`, `Price`, `Customer ID`, `Country` 等欄位，背景為一家總部位於英國、專門銷售禮品的線上零售商。

您可以將這些情境貼入 `抽象需求轉換.md` 的 Prompt 中，練習如何將這些業務直覺轉化為基於該數據集的具體 SQL/Python 分析任務。

---

## 案例一：尋找沈睡的鯨魚 (CRM 經理)

**📖 情境故事 (Background Story)**:
你是一家主打獨特禮品的電商平台「GiftHub」的 CRM 經理。公司過去兩年累積了大量交易數據，但行銷活動一直是「亂槍打鳥」。老闆希望能區分出「批發商大戶 (Whales)」和「偶爾買的小資族 (Minnows)」。更重要的是，你的業績壓力來自於「喚醒」那些曾經買很多但最近消失的 VIP。你手上有 `Customer ID` 和他們的交易紀錄，你需要透過數據將他們分群，即刻啟動精準行銷。

**👉 請複製以下內容填入 Prompt**:

```markdown
# CONTEXT (背景資訊)
- **我的角色**: 電商平台 CRM 經理 (Customer Relationship Manager)
- **原始商業需求**: 
"我們的客戶資料庫裡有幾千個 `Customer ID`，但我完全不知道誰才是真正的金雞母。
有些客戶好像很久沒回來了（Recency），以前買的頻率很高（Frequency），每次下單金額也都很大（Monetary）。
我希望能用這種 RFM 的邏輯把客戶分成幾個等級，例如『沈睡的VIP』或『潛力新客』。
請幫我定義出這些客群，並告訴我『沈睡VIP』大概有多少人？他們貢獻了多少營收？我想針對他們發送回歸優惠券。"
```

## 案例二：倉庫大掃除 (商品採購總監)

**📖 情境故事 (Background Story)**:
你是商品採購總監。聖誕節快到了，但倉庫裡堆滿了超過 4,000 種不同的商品 (`StockCode`)，空間嚴重不足。你懷疑其中有大量商品是「叫好不叫座」或根本是「滯銷品」。你需要依據二八法則 (Pareto Principle)，找出那些貢獻了 80% 利潤的少數「明星商品」，同時列出那些佔用空間卻賣不動的「殭屍商品」清單，準備進行清倉拍賣或直接下架。

**👉 請複製以下內容填入 Prompt**:

```markdown
# CONTEXT (背景資訊)
- **我的角色**: 商品採購總監 (Merchandising Director)
- **原始商業需求**: 
"倉庫快爆炸了，我需要做一次徹底的 SKU 健檢。
我有幾千個 `StockCode`，但我憑直覺認為只有少數商品在賺錢。
幫我用數據跑一次 ABC 分析（或是帕累托分析）：
1. 找出營收貢獻前 20% 的核心商品，這些絕對不能斷貨。
2. 揪出那些過去一年銷量（Quantity）極低，或者被退貨（Cancellation）很多次的『雷品』。
我需要一份『建議淘汰清單』，讓我能有憑有據地把它們清掉。"
```

## 案例三：誰在一直退貨？ (營運優化經理)

**📖 情境故事 (Background Story)**:
你是負責營運效率的經理。在審視財報時，發現「退貨與折讓」的成本比例異常升高。在資料庫中，退貨訂單的 `InvoiceNo` 通常是以 'C' 開頭，且 `Quantity` 為負值。你需要像偵探一樣，找出這波退貨潮的源頭。是特定商品 (`StockCode`) 品質有問題？還是特定國家 (`Country`) 的物流頻繁損壞？抑或是某些 `Customer ID` 惡意大量退貨？

**👉 請複製以下內容填入 Prompt**:

```markdown
# CONTEXT (背景資訊)
- **我的角色**: 營運優化經理 (Operations Manager)
- **原始商業需求**: 
"我看到系統裡有一大堆 `InvoiceNo` 是 'C' 開頭的訂單，這些都是退貨，對利潤很傷。
我想知道問題出在哪？
是我們有些產品（Description）本身質量太差，導致退貨率高到離譜？
還是特定國家（例如除了英國以外的地區）因為物流太久導致客人拒收？
甚至，有沒有哪些客戶（Customer ID）是『奧客』，買了又一直退？
幫我找出退貨率最高的幾個『罪魁禍首』，我要採取行動截斷止血。"
```
