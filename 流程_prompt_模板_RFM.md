# 端到端流程 Prompt 模板：從問題定義到 Streamlit 儀表板上線（以 RFM 客群分析為例）

本模板整理你這次完整走過的流程，日後只要換資料集，依照各階段的 Prompt 與產出，一步步替換即可。

建議資料夾結構（已存在）：  
`00問題定義研究 → 01資料描述 → 02數據分析 → 03圖表選用 → 04報告呈現 → 儀錶板系統上線_streamlit`

---

## 0. 問題定義研究（00問題定義研究）

**目標**：把「模糊需求」轉成「可分析的問題定義書」。

### 0-1 抽象需求轉換

對 ChatGPT/助手下的 Prompt 範例：

> 參考 `00問題定義研究/抽象需求轉換.md` 的模板，\  
> 針對以下商業情境：「[描述你的新情境，例如：訂閱制 SaaS 想提升續約率]」，\  
> 幫我產出一份「抽象需求轉換」說明，整理出潛在數據問題與可行分析方向。

### 0-2 產出分析問題定義書

> 參考 `00問題定義研究/商業場景模擬案例.md` 與 `抽象需求轉換 1.md` 的風格，\  
> 幫我針對上述情境，產出一份 `分析問題定義書`（Markdown），\  
> 放在 `00問題定義研究/分析問題定義書_?.md`，內容包含：  
> 1. 核心問題  
> 2. 分析目標  
> 3. 關鍵指標  
> 4. 初步假設  
> 5. 所需數據  
> 6. 預期產出與商業價值

---

## 1. 資料描述與數據卡（01資料描述）

**目標**：用「數據卡」把新資料集講清楚，後面分析與溝通都引用這份說明。

### 1-1 產出數據卡

> 參考 `01資料描述/data_card_prompt.md`，\  
> 對新的資料集（路徑：`[你的新檔案路徑]`）產出一份數據卡，\  
> 放到 `01資料描述/[資料集名稱]_數據卡.md`，\  
> 內容包含欄位解釋、缺失值、異常值、初步品質評估與探索建議。

---

## 2. 數據分析與模型選擇（02數據分析）

**目標**：根據問題定義書，決定分析策略，寫 Python 分析腳本並產出洞察。

### 2-1 分析策略建議（選模型 / 框架）

> 參考 `02數據分析/商用模型思維.md` 的模板，\  
> 以 `00問題定義研究/分析問題定義書_?.md` 為依據，\  
> 幫我產出一份 `分析策略建議_?.md`，\  
> 指定 1–2 個最適合的分析框架/模型（例如 RFM、分群、迴歸…），\  
> 並說明推薦理由、分析步驟、預期洞察。

（本次案例：選用「RFM 客戶分群 + 沈睡 VIP 量化」。）

### 2-2 Python 數據分析腳本

> 依 `分析策略建議_?.md`，\  
> 幫我撰寫 Python 腳本 `02數據分析/[分析主題]_analysis.py`，\  
> 需求：  
> - 讀取新資料集路徑  
> - 做資料清洗（依數據卡建議）  
> - 依策略計算所需指標（本例為 R、F、M）  
> - 產出彙總表（例如 `rfm_segment_summary.csv`）  
> - 產出名單表（例如 `sleep_vip_list.csv`）  
> - 印出關鍵摘要（方便在終端查看）

### 2-3 關鍵數據洞察報告

> 參考 `02數據分析/關鍵數據洞察_Online_Retail_RFM.md` 的結構，\  
> 用剛剛的輸出 CSV（例如 `rfm_segment_summary.csv`、名單檔）\  
> 產出 `關鍵數據洞察_[新主題].md`，\  
> 內容至少包含：  
> - 有效分析樣本定義（明細 vs 不重複客戶）  
> - 各區隔規模與營收貢獻表  
> - 針對關鍵群體（如沈睡 VIP）的核心指標  
> - 驗證分析問題定義書中的假設  
> - 對應的文字洞察與建議

---

## 3. 圖表選用與視覺化設計（03圖表選用）

**目標**：把洞察對應到合適的圖表類型，寫出設計建議（之後 Python 或 BI 工具都能實作）。

### 3-1 圖表設計建議

> 參考：  
> - `03圖表選用/圖表類型參考.md`  
> - `03圖表選用/圖表選型.md`  
> - `03圖表選用/色系風格.md`（例如深海洞察配色）\  
> 以 `關鍵數據洞察_[新主題].md` 的重點洞察為基礎，\  
> 幫我產出一份 `圖表設計建議_[新主題].md`（可仿本次 `圖表設計建議_v2.md` 結構），\  
> 對每個洞察說明：  
> - 推薦圖表類型（比較/構成/分佈/關聯）  
> - 圖表標題、軸設計  
> - 顏色與排序建議  
> - 替代方案

---

## 4. 靜態報告呈現（04報告呈現）

**目標**：用金字塔原則整理成一份可以直接給主管看的分析報告，並產出對應圖檔。

### 4-1 報告骨架與內容

> 參考 `04報告呈現/分析報告製作.md` 的模板，\  
> 以 `關鍵數據洞察_[新主題].md` 與 `圖表設計建議_[新主題].md` 為依據，\  
> 幫我產出 `04報告呈現/分析報告_[新主題].md`，\  
> 結構包含：  
> - 執行摘要  
> - 關鍵發現（對應每張關鍵圖）  
> - 詳細分析  
> - 商業建議  
> - 附錄（數據來源、方法、產出檔案清單）

### 4-2 Python 產出靜態圖檔（選配）

> 根據 `圖表設計建議_[新主題].md`，\  
> 幫我撰寫 `04報告呈現/generate_report_charts_[新主題].py`，\  
> 使用 Matplotlib 或 Plotly，讀取 `02數據分析` 的 CSV，\  
> 產出對應的 PNG 圖檔（放在 `04報告呈現/`），\  
> 並在報告中用檔名引用。

## 5. 儀表板設計藍圖與 Streamlit 上線（儀錶板系統上線_streamlit）

**目標**：把靜態報告與圖表設計，升級成「可互動儀表板」，並有清楚的上線流程。

### 5-1 儀表板藍圖

> 參考 `儀錶板系統上線_streamlit/動態儀表板藍圖_RFM沈睡VIP.md`，\  
> 以 `分析報告_[新主題].md` 的關鍵發現與商業建議為基礎，\  
> 幫我產出 `動態儀表板藍圖_[新主題].md`，\  
> 內容包括：  
> - 儀表板類型與 North Star Metric  
> - 佈局與 UX（F 型佈局）  
> - 組件設計表（KPI 卡、圖表、名單表…）  
> - 篩選器與互動性  
> - 與報告/圖表設計建議的對應關係

### 5-2 Streamlit App 生成

> 參考 `Streamlit_App_Generator_Prompt.md`，\  
> 以 `動態儀表板藍圖_[新主題].md` 與 `02數據分析` 的 CSV 為輸入，\  
> 幫我生成 `儀錶板系統上線_streamlit/app.py`：  
> - 使用 `streamlit`, `pandas`, `plotly.express`  
> - 建立 `@st.cache_data` 的資料載入函式  
> - 每個圖表/區塊封裝成函式  
> - 依藍圖實作佈局與篩選器  
> - 讀取 `02數據分析` 的彙總表與名單表

### 5-3 啟動說明與 uv 虛擬環境

> 參考 `啟動說明書_Streamlit_RFM.md`，\  
> 幫我為新主題寫一份啟動說明書 `啟動說明書_Streamlit_[新主題].md`，\  
> 說明：  
> - 如何用 uv 在根目錄執行分析腳本產出 CSV  
> - 如何用 uv 或既有虛擬環境啟動 Streamlit  
> - 主要指令範例（`uv run python ...`、`uv run streamlit run app.py`）

## 6. 快速複用清單（換資料集時檢查）

1. **是否已有新的分析問題定義書？** → 若沒有，回到 §0。  
2. **是否為新資料集產出數據卡？** → §1。  
3. **分析策略是否需要換模型？** → 更新 §2 分析策略與 Python 腳本。  
4. **彙總表與名單檔是否更新？** → `02數據分析` 下的 CSV。  
5. **圖表設計是否需調整（類別數、指標改變）？** → 更新 §3 圖表設計建議。  
6. **報告與儀表板是否需同步更新？** → 重跑 §4–§5 的 Prompt 與腳本。
